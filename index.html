colorspaces.js

A tiny jQuery library for manipulating colors.

var crimson = $.colorspaces.make_color('hex', '#dc143c');
var LCH = crimson.as('CIELCH');

RGB, the color space we are forced to use here on the web is based on display technology, not human color perception. For example, the "lightness" parameter in the HSL representation of RGB is rather bad at describing lightness:

for (var h = 0; h < 360; h += 30) {
  var square = $('<div class="square">a</div>').appendTo($('#hsl_demo'));
  square.css('background-color', 'hsl(' + h + ', 80%, 50%)');
}

Notice how the red and the blue square are very noticeably darker than, say, the yellow even though all of them have the same lightness. Compare this to the results given by a similar color space, CIE LCH:

for (var h = 0; h < 360; h += 30) {
  var square = $('<div class="square">a</div>').appendTo($('#lch_demo'));
  var color = $.colorspaces.make_color('CIELCH', [75, 100, h]);
  square.css('background-color', color.as('hex'));
}

Currently, colorspaces.js supports sRGB, CIE XYZ, CIE xyY, CIE L*a*b* and CIE LCh color spaces.

colorspaces.make_color takes two arguments: the name of the color space ('sRGB', 'CIEXYZ', 'CIExyY', 'CIELAB', 'CIELCH') and an array of values representing the color in the color space. The returned object has one method, as, that takes the name of a color space as its only argument and returns the coordinates of the color in that color space as an array.

You can use 'hex' in place of a color space name to parse or output a hex RGB representation of a color.
